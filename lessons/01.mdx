# HTML

- **H**yper**T**ext **M**arkup **L**anguage 
- Created by Tim Berners-Lee to enable document sharing (text based -> links, headings, paragraphs)
- **Structure of a Web page**
- Set of HTML elements, telling browser how to display content
- HTML 5 is latest specification (each version brings new tags and depricates others) -> W3C
- `<img src="path_to_image" alt="Text if image fails to load" />`
- `<b>This is bold text</b>`

<Notes>

- HyperText -> text with referecnes to other pages (links)
- Markup -> standardized set of notations (tags and attributes)
- Language -> not programming language (it's more of a representation or format)
- **tag** is `img`, **attribute** is `src` and `<img src="URL" />` is **element**
- [W3C reference](https://www.w3schools.com/html/default.asp)
- [HTML validator](https://validator.w3.org/nu/#textarea)
- Each HTML document has `html`, `body`, `head` tags.

</Notes>

# CSS

- **C**ascading **S**tyle **S**heets
- How HTML elements are displayed on device
- CSS3 specification
- `color: blue;`

<Notes>

- Cascading -> styles cascade (apply to lower levels) if not overridden
- Style Sheets -> suggests separate CSS file(s), but CSS could also be inline, or embedded in `style` tag
- `color` is property, `blue` is value, don't forget semicolon (if ommitted it would merge with next line and then both lines would be invalid and not applied)

</Notes>

---

# CSS Selectors

- Select all elements with the same tag
  - `h2 { text-transform: "uppercase"; }`
- select all elements with the same `class` attribute
   - `.className { background-color: "tomato"; }`
- Select all elements with the same `id` attribute
   - `#uniqueId { text-align: "center"; }`
- Selectors can be mixed
   - `h2.specialHeading { transform: rotate(180deg); }`
   - `h2 .specialHeading { transform: rotate(180deg); }`
- Universal selector (`*`), Grouping selector (`div, p { ... }`)

<Notes>

- Selecting tags could be useful for resetting default browser styling (i.e. `ul` elements)
- Classes are most common CSS selectors
- Id selectors are used sparingly. to style unique elements
- Id attributes should be unique for each element and should appear only once on each page. 
Plural form is used because it would work on multiple elements with the same `id`, but this is an anti-pattern and should be avoided.
- Mixing selctors is useful when certain styles need to be applyed only in specific context (e.g. styling inputs in forms differently)

</Notes>

# Adding CSS

- `<link rel="stylesheet" type="text/css" href="myStyleSheet.css">`
- Inline style > Id Styling > Class styling > Tag styling


<Notes>

- When adding multiple stylesheets, they are applied in order they are linked. 
Common mistake is thinking that styles are applyed based on their order on element. 
It's actually their link order and order in file where they are defined. This all applies for the same level of selector (e.g. classes)

</Notes>

# CSS position

- Type of positioning used for an element
- `relative` -> relative to element position (`top`, `left`, ...)
- `fixed` -> realtive to a screen (**not** content on the screen)
- `absolute` -> relative to nearest positioned ancestor

<Notes>

- `position: static;` is default
- `fixed` -> relative to a screen as hardware, when you scroll screen doesn't move, content moves  
- positioned ancestor -> any element with position property different then `static` (default)
- `absolute` -> commonly used with `relative` parent, `z-index` property changes if elements are before or after
- `sticky` is new position property, it's positioning depends on scroll position (absolute or fixed)
- [More on positioning](https://www.w3schools.com/css/css_positioning.asp)

</Notes>

---

# CSS transitions

- Allows smooth changing of property value over some time
- `transition: opacity 0.5s ease-in-out;`

<Notes>

- property, period, time curve -> transition syntax
- works on exact values only, start and end values must be known before transition.
Common bug is transitioning height from 0px to 100%, which doesn't work because browser doesn't know value for 100%.

</Notes>

--- 

# CSS Media

- CSS2 - different style rules for different media (screens, printers)
- CSS3 - different style rules for different device capatibilities (resolution, orientation)
- responsive design -> different CSS for different resolutions 
- `@media screen and (min-width: 400px) { ...will apply only on screens wider than 400px }`

<Notes>

- today developers take mobile first approach -> first they write CSS for smaller screens, 
then they add media queries to modify styles for wider devices

</Notes>
